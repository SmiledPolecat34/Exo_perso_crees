Exercice de montée en compétences sur Linux Server

    Objectifs:

  • Installer et configurer un serveur web Nginx
  • Gérer les utilisateurs et les permissions
  • Configurer une tâche planifiée
  • Sécuriser le serveur avec UFW (Uncomplicated Firewall)
  • Rechercher des fichiers et gérer l'espace disque
    • Instructions détaillées:
  • Installation de Nginx:
    • Installez le serveur web Nginx.
    • Vérifiez que Nginx est actif et s'exécute au démarrage du système.
    • Création et gestion des utilisateurs:
    • Créez un utilisateur système nommé devuser avec /usr/sbin/nologin comme shell par défaut, et un répertoire d'accueil fictif.
    • Ajoutez cet utilisateur au groupe www-data.
    Configuration de Nginx:
  • Configurez un nouveau site virtuel sous Nginx. Utilisez /var/www/monsite comme racine du document et monsite.local comme nom de serveur.
  • Assurez-vous que le site est accessible en modifiant le fichier /etc/hosts sur le client.
    • Gestion des permissions:
    • Changez le propriétaire du dossier /var/www/monsite pour que devuser soit le propriétaire et www-data le groupe.
    • Mettez en place des permissions permettant à devuser d'écrire dans le dossier, tout en permettant à Nginx de servir le contenu.
    Configuration d'UFW (Uncomplicated Firewall):
  • Activez UFW et configurez-le pour autoriser uniquement le trafic sur les ports 80 (HTTP) et 443 (HTTPS), ainsi que votre nouveau port SSH.
  • Planification d'une tâche avec cron:
  • Créez une tâche cron pour devuser qui nettoie le dossier /tmp tous les jours à minuit.
    • Recherche de fichiers:
  • Utilisez la commande find pour rechercher tous les fichiers modifiés dans les dernières 24 heures dans le répertoire /var/www.
    • Gestion de l'espace disque:
  • Trouvez les cinq dossiers qui prennent le plus d'espace sur votre serveur et listez-les avec leur taille.
    • Livrables:
  • Le fichier de configuration de votre site Nginx.
  • La commande utilisée pour créer devuser et le détail des permissions appliquées à /var/www/monsite.
  • La règle UFW pour limiter l'accès aux ports nécessaires.
  • La ligne de commande cron pour nettoyer /tmp.
  • Les commandes utilisées pour la recherche de fichiers et la gestion de l'espace disque, ainsi que leurs sorties.
